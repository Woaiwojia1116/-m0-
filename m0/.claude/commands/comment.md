## 命令说明
这是一个本地代码审查命令，不依赖 GitHub、Git 或 gh CLI。

## 使用方式
用户输入 `/review` 或 "帮我审查代码" 时，按以下规则执行：

## 第一步：确认审查范围
向用户询问：
- "请指定要审查的文件（支持多选，如 `PID.c, PID.h` 或直接说'全部'）"
- "代码运行的平台是什么？（如 STM32F103 / Arduino Uno / Linux x86）"
- "有没有特别关注的点？（如刚修复了一个 bug / 担心性能问题 / 想检查安全性）"

## 第二步：主动读取文件
使用 `read_file` 工具读取用户指定的本地文件。如果用户说"全部"，读取当前目录下所有 `.c` 和 `.h` 文件。

## 第三步：执行审查
按以下 checklist 逐项检查：

### 必查项（优先级从高到低）
1. [ ] 是否有数组越界风险？
2. [ ] 是否有未初始化的变量？
3. [ ] 是否在中断和主循环中访问共享变量而没有保护？
4. [ ] `volatile` 是否正确使用？
5. [ ] 延时函数是阻塞式还是非阻塞式？
6. [ ] PID 增量式/位置式实现是否有积分饱和问题？
7. [ ] 串口接收是否有溢出风险？是否使用环形缓冲区？
8. [ ] 电机控制是否有占空比限幅？
9. [ ] 除法/取模是否在校验除数非零之后？
10. [ ] 函数是否有清晰的输入输出范围说明？

### 选查项
- [ ] 函数长度是否超过 100 行？
- [ ] 是否有深层嵌套（超过 4 层）？
- [ ] 全局变量是否能改为局部 + 传参？
- [ ] 注释是否与代码同步？

## 第四步：输出审查报告

采用以下格式：
╔════════════════════════════════════════════════════╗
║ 代码审查报告 ║
║ 文件数: X | 严重: X | 一般: X | 建议: X ║
╚════════════════════════════════════════════════════╝
